# pars

Инструмент для полной выгрузки каталога https://www.elcats.ru/ в SQLite-базу данных.

## Установка

```bash
pip install -r requirements.txt
```

## Запуск парсера

```bash
python -m elcats --output data/elcats.db --delay 0.2 --workers 4
```

Параметры:

- `--output` — путь до SQLite файла (по умолчанию `data/elcats.db`).
- `--delay` — пауза между запросами в секундах.
- `--workers` — число потоков для параллельной обработки модификаций (по умолчанию 1).
- `--brand` — ограничение по бренду (можно указывать несколько раз).
- `--log-level` — уровень логирования (`INFO`, `DEBUG` и т.д.).

Парсер сохраняет данные в таблицы с полями:

- **brands** — бренды каталога.
- **vehicles** — модификации/модели для бренда.
- **party_categories** — группы и подгруппы деталей.
- **parts** — детали из выбранных подгрупп.
- **parts_to_vehicles** — связи деталей с транспортными средствами.
- **attributes** — дополнительные свойства деталей и подгрупп (например, количество, примечания, коды и т.п.).
- **scrape_progress** — служебная таблица, где фиксируется состояние выгрузки по каждой модификации.

Повторный запуск парсера автоматически продолжит работу с последней незавершённой модификации: уже обработанные сочетания бренд/модель/модификация будут пропущены, поэтому повторно выгружать весь каталог не потребуется.

> ⚠️ Полный сбор данных по всем брендам занимает значительное время (десятки тысяч запросов). При необходимости можно запускать парсер по одному бренду.
