# pars

Инструмент для полной выгрузки каталога https://www.elcats.ru/ в SQLite-базу данных. Ниже — максимально подробная инструкция «с нуля».

## 1. Что понадобится

1. **Современный Python (3.11 или новее).**
   - Windows: скачайте установщик с [python.org](https://www.python.org/downloads/), при установке включите пункт «Add Python to PATH» и оставьте стандартные параметры.
   - macOS: установите через официальный пакет или `brew install python@3.11`.
   - Linux: установите из репозитория (`sudo apt install python3 python3-venv python3-pip` и т.п.).
2. **Свободное место на диске** — несколько гигабайт под временную базу и итоговые данные.
3. **Стабильное интернет-соединение** — выгрузка может идти долго.

## 2. Получите проект

1. Скачайте архив с кодом или клонируйте репозиторий:
   ```bash
   git clone <ссылка-на-репозиторий>
   ```
2. Распакуйте архив (если скачивали) и откройте папку проекта в проводнике.

## 3. (Необязательно) Создайте изолированную среду

Это поможет не смешивать библиотеки парсера с другими программами.

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
```

После завершения работы введите `deactivate`, чтобы выйти из виртуального окружения.

## 4. Установите зависимости

Откройте терминал/PowerShell в папке проекта и выполните:

```bash
pip install -r requirements.txt
```

Python скачает и установит нужные библиотеки (HTTP-клиент, парсер HTML и т.д.). Делается один раз.

## 5. Запустите парсер

Базовая команда (можно скопировать как есть):

```bash
python -m elcats --output data/elcats.db --delay 0.5 --workers 4
```

Что произойдёт:

1. Создастся папка `data` (если её нет) и файл SQLite `elcats.db` с таблицами каталога.
2. Скрипт пойдёт по всем брендам и моделям, сохраняя детали и связи.
3. Прогресс автоматически фиксируется: при остановке или сбое следующий запуск продолжит с места, где всё оборвалось.

### Часто используемые параметры

| Параметр | Что делает | Значение по умолчанию |
| --- | --- | --- |
| `--output PATH` | Куда сохранить SQLite-файл. Укажите полный путь, если нужно положить в другое место. | `data/elcats.db` |
| `--delay SECONDS` | Пауза между сетевыми запросами. Увеличивайте, если сайт отвечает ошибками 429/503. | `0.2` |
| `--workers N` | Сколько потоков запускать параллельно. Рекомендовано 2–6 (слишком много увеличивает нагрузку). | `1` |
| `--brand NAME` | Ограничить выгрузку одним брендом. Можно указать несколько раз: `--brand kia --brand hyundai`. | все бренды |
| `--log-level LEVEL` | Детальность логов (`INFO`, `DEBUG`, `WARNING`). | `INFO` |

Посмотреть справку по всем опциям можно командой `python -m elcats --help`.

## 6. Контроль процесса

* Логи выводятся прямо в консоль. Чтобы сохранить их в файл, используйте перенаправление:  
  `python -m elcats ... > elcats.log 2>&1`
* Если работа прервалась (например, закрыли окно, отключился интернет), просто запустите ту же команду снова — обработанные модификации будут пропущены.

## 7. Что находится в базе

* **brands** — список брендов.
* **vehicles** — модели и модификации.
* **party_categories** — иерархия групп и подгрупп.
* **parts** — детали с описанием.
* **parts_to_vehicles** — связь деталей с модификациями.
* **attributes** — дополнительные данные (коды, количества, примечания).
* **scrape_progress** — служебный журнал, который позволяет возобновлять работу.

Откройте файл `elcats.db` любым просмотрщиком SQLite (DB Browser for SQLite, DBeaver, TablePlus) и экспортируйте таблицы в нужный формат или перенесите в свою основную базу напрямую.

## 8. Советы по производительности

* Начинайте с консервативной задержки (`--delay 0.5`). Если сервер отвечает стабильно, можно немного уменьшать, следя за ошибками.
* Не поднимайте `--workers` слишком высоко, чтобы не перегружать сайт и не получить блокировку.
* При повторной выгрузке можно ограничиться конкретными брендами — так экономится время.

## 9. Типичные проблемы

| Проблема | Решение |
| --- | --- |
| `ModuleNotFoundError` при запуске | Убедитесь, что активирована виртуальная среда и выполнено `pip install -r requirements.txt`. |
| Ошибки 429/503 в логах | Увеличьте задержку (`--delay 1.0` или больше) и снизьте число потоков. |
| Нужно начать заново с чистой базы | Удалите файл `elcats.db` и запустите парсер ещё раз — он создаст новую базу. |

После выгрузки файл SQLite можно хранить как резервную копию или импортировать данные в любую другую СУБД.
